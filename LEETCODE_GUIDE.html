Структури даних, паттерни та підходи до вирішення алгоритмічних задач. З прикладами на Kotlin.

Що таке LeetCode і навіщо він потрібен

LeetCode — це платформа з алгоритмічними задачами. Їх вирішують на технічних інтерв'ю в Google, Meta, Amazon, Spotify та інших компаніях.

Головна ідея: навчитись розпізнавати паттерни і знати яку структуру даних застосувати в конкретній ситуації. Не вчити відповіді — а вчити підходи.

Big O — складність алгоритму

Big O описує наскільки швидко росте час виконання при збільшенні вхідних даних n.

O(1) швидко
Константа. Читання з масиву за індексом.
O(log n) швидко
Логарифм. Бінарний пошук.
O(n) нормально
Лінійний. Один прохід по масиву.
O(n log n) нормально
Сортування. Merge sort, heap sort.
O(n²) повільно
Квадратичний. Вкладені цикли.
O(2ⁿ) дуже повільно
Експоненціальний. Всі підмножини.
Правило: якщо n = 10⁵, то O(n²) = 10¹⁰ операцій — це занадто повільно. Прагни до O(n) або O(n log n).

Структури даних

Array (масив)

Елементи в пам'яті йдуть підряд. Доступ за індексом — миттєво.

val arr = intArrayOf(1, 2, 3, 4, 5)
arr[0]       // читання — O(1)
arr[2] = 10  // запис — O(1)

Коли використовувати: індексований доступ, sliding window, two pointers.

HashMap / HashSet

HashMap зберігає пари ключ → значення. HashSet зберігає унікальні елементи. Пошук і вставка — O(1).

val map = mutableMapOf<Int, Int>()
map[5] = 10          // зберегти
map[5]               // отримати — 10
map.containsKey(5)   // перевірити — true

val set = mutableSetOf<Int>()
set.add(3)
set.contains(3)      // true

Коли використовувати: треба швидко перевірити "чи бачили ми це вже" або "скільки разів зустрічалось".

Приклад — Two Sum: знайти два індекси, сума яких = target.

fun twoSum(nums: IntArray, target: Int): IntArray {
    val seen = mutableMapOf<Int, Int>() // значення -> індекс
    for ((i, num) in nums.withIndex()) {
        val complement = target - num
        if (complement in seen) {
            return intArrayOf(seen[complement]!!, i)
        }
        seen[num] = i
    }
    return intArrayOf()
}

Stack (стек)

Принцип LIFO — Last In, First Out. Як стопка тарілок.

val stack = ArrayDeque<Int>()
stack.addLast(1)   // push
stack.addLast(2)
stack.removeLast() // pop — повертає 2
stack.last()       // peek — дивимось без видалення

Коли використовувати: дужки, наступний більший елемент, відміна дій.

Приклад — Valid Parentheses:

fun isValid(s: String): Boolean {
    val stack = ArrayDeque<Char>()
    for (c in s) {
        when (c) {
            '(', '[', '{' -> stack.addLast(c)
            ')' -> if (stack.isEmpty() || stack.removeLast() != '(') return false
            ']' -> if (stack.isEmpty() || stack.removeLast() != '[') return false
            '}' -> if (stack.isEmpty() || stack.removeLast() != '{') return false
        }
    }
    return stack.isEmpty()
}

Linked List (зв'язний список)

Кожен вузол містить значення і посилання на наступний вузол.

class ListNode(var `val`: Int, var next: ListNode? = null)

Класичний трюк — Fast & Slow Pointers:

// Знайти середину списку
fun middleNode(head: ListNode?): ListNode? {
    var slow = head
    var fast = head
    while (fast?.next != null) {
        slow = slow?.next
        fast = fast.next?.next  // fast рухається вдвічі швидше
    }
    return slow // slow зупинився на середині
}

Tree (дерево)

class TreeNode(var `val`: Int, var left: TreeNode? = null, var right: TreeNode? = null)

DFS — обхід в глибину (рекурсія, йде вглиб до листа і повертається):

fun inorder(root: TreeNode?): List<Int> {
    if (root == null) return emptyList()
    return inorder(root.left) + root.`val` + inorder(root.right)
}

BFS — обхід в ширину (черга, рівень за рівнем):

fun levelOrder(root: TreeNode?): List<List<Int>> {
    if (root == null) return emptyList()
    val result = mutableListOf<List<Int>>()
    val queue = ArrayDeque<TreeNode>()
    queue.addLast(root)
    while (queue.isNotEmpty()) {
        val level = mutableListOf<Int>()
        repeat(queue.size) {
            val node = queue.removeFirst()
            level.add(node.`val`)
            node.left?.let { queue.addLast(it) }
            node.right?.let { queue.addLast(it) }
        }
        result.add(level)
    }
    return result
}

Heap / PriorityQueue

Завжди тримає мінімальний (або максимальний) елемент зверху. Вставка і видалення — O(log n).

val minHeap = PriorityQueue<Int>()                   // мін зверху
val maxHeap = PriorityQueue<Int>(reverseOrder())       // макс зверху

minHeap.add(5)
minHeap.add(1)
minHeap.add(3)
minHeap.poll() // 1 — завжди повертає мінімум

Коли використовувати: "знайди K найбільших/найменших елементів".

Паттерни вирішення задач

1
Two Pointers
Коли: відсортований масив, пари, palindrome

Два вказівники рухаються по масиву — один з початку, один з кінця (або обидва з початку).

// Чи є рядок паліндромом?
fun isPalindrome(s: String): Boolean {
    var left = 0
    var right = s.length - 1
    while (left < right) {
        if (s[left] != s[right]) return false
        left++
        right--
    }
    return true
}
2
Sliding Window
Коли: підрядок, підмасив, максимум в діапазоні

Вікно фіксованого або змінного розміру ковзає по масиву. Дозволяє уникнути вкладених циклів.

// Максимальна сума підмасиву розміру k
fun maxSumSubarray(nums: IntArray, k: Int): Int {
    var windowSum = nums.take(k).sum()
    var maxSum = windowSum
    for (i in k until nums.size) {
        windowSum += nums[i] - nums[i - k] // додаємо новий, прибираємо старий
        maxSum = maxOf(maxSum, windowSum)
    }
    return maxSum
}
3
Binary Search
Коли: відсортований масив, пошук мінімального значення при умові

Щоразу відкидаємо половину варіантів. Застосовується не тільки для пошуку числа — але і для пошуку відповіді в діапазоні.

fun binarySearch(nums: IntArray, target: Int): Int {
    var left = 0
    var right = nums.size - 1
    while (left <= right) {
        val mid = left + (right - left) / 2 // так не буде overflow
        when {
            nums[mid] == target -> return mid
            nums[mid] < target -> left = mid + 1
            else               -> right = mid - 1
        }
    }
    return -1
}
4
Dynamic Programming (DP)
Коли: кількість способів, максимум/мінімум вартості, чи можливо досягти

Розбиваємо задачу на підзадачі і зберігаємо їх результати щоб не рахувати двічі.

Шаблон думки:
1. Що таке dp[i]? (що воно означає)
2. Яка базова умова? (dp[0] = ?)
3. Яка формула переходу? (dp[i] = dp[i-1] + ...) 
// Fibonacci — класичний DP
fun fib(n: Int): Int {
    if (n <= 1) return n
    val dp = IntArray(n + 1)
    dp[0] = 0
    dp[1] = 1
    for (i in 2..n) {
        dp[i] = dp[i - 1] + dp[i - 2]
    }
    return dp[n]
}
5
Backtracking
Коли: всі комбінації, всі перестановки, sudoku

Перебираємо всі можливості, повертаємось якщо зайшли в глухий кут.

// Всі підмножини масиву
fun subsets(nums: IntArray): List<List<Int>> {
    val result = mutableListOf<List<Int>>()
    fun backtrack(start: Int, current: MutableList<Int>) {
        result.add(current.toList())       // зберігаємо поточний стан
        for (i in start until nums.size) {
            current.add(nums[i])
            backtrack(i + 1, current)
            current.removeLast()           // відкочуємось назад
        }
    }
    backtrack(0, mutableListOf())
    return result
}

Як підходити до задачі

  1. Прочитай умову двічі. Виділи що дано і що треба повернути.
  2. Розбери приклади вручну на папері. Намалюй масив, дерево, список.
  3. Запитай себе: масив відсортований? → Binary Search. Підрядок? → Sliding Window. Дерево? → DFS/BFS. Кількість способів? → DP.
  4. Напиши brute force рішення (навіть O(n²)). Краще повільне але правильне.
  5. Оптимізуй. Чи можна уникнути повторних обчислень? Чи потрібна HashMap?
  6. Перевір edge cases: пустий масив, один елемент, від'ємні числа, нулі.
